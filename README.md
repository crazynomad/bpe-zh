# BPE（Byte Pair Encoding）分词可视化 · 字节级

一个交互式动画，从零演示 **字节级 BPE（Byte-Level Byte Pair Encoding）** 的训练过程，
帮你直观看懂一个常被忽略的事实：**中文为什么比英文更费 token。**

🔗 在线体验：<https://crazynomad.github.io/bpe-zh/>

> 灵感来自 [bpe-visualizer.com](https://www.bpe-visualizer.com/)（字符级）。本项目改做**字节级**，
> 以贴近 GPT-2 / GPT-4 等真实分词器，并针对中文补充了"字节 ↔ 汉字"双视图。
> 视觉套用绿皮火车 **Stage Dusk** 设计系统（暗舞台 · 霓虹青 / AI 蓝 / 琥珀 · JetBrains Mono）。

> "可视化的目的是洞察，而非图像。" —— Ben Shneiderman

## 为什么要做字节级？

大多数 BPE 演示是**字符级**的：把文本按字符切开再合并。但真实的大模型分词器走的是**字节级**：

1. 先把文本 **UTF-8 编码成字节**；
2. 在字节序列上统计相邻对频率、反复合并最高频的对。

关键差异在中文上暴露无遗：

| 文本 | UTF-8 字节数 | 起始 token 数 |
|------|------------|--------------|
| `cat` | 3 | 3 |
| `猫`  | 3 | **3**（一个汉字 = 3 字节） |
| `你好` | 6 | 6 |

英文一个字母就是 1 字节、1 个起始 token；**一个汉字要占 3 个字节**，分词器得先把这 3 个裸字节
"拼回"成一个汉字，才谈得上进一步合并。这就是中文 token 消耗更高的根源。

## 功能

- **可直接编辑的原文输入** —— 在「原文」面板里直接打字 / 粘贴，或点选内置**名篇范本**：
  I Have a Dream（英 / 双语）、乔布斯经典名句、Attention Is All You Need 论文全文、
  刘家琨普利兹克奖获奖感言（双语）、朱自清《春》。
- **分步播放** —— 播放 / 暂停 / 单步 / 倍速；进度条；可点击的**全宽步骤条**（每步显示合并式、
  出现次数、token 数、词表大小），只展示当前及之前的步骤。
- **三栏可视化** —— 原文 ｜ 当前 token 序列 ｜ 当前词表 + 本步生成 token。
- **统计面板** —— 原始字节数、当前 token 数、压缩率、词表大小。
- **字符编码视图（可开关）** —— 勾选后原文切到只读的逐字符方格，显示每个字符的 **Unicode 码位**
  与字节数；无文字时禁用。
- **N / K 双计数** —— 区分"合并前相邻对频次 N"与"合并后新 token 实际出现次数 K"，并解释二者
  在周期性重叠时的差异。
- **响应式** —— 窄屏堆叠，≥1024px 三栏并排，支持放大到 2560×1440 等超宽屏。

## 可视化怎么读

- **原文**：默认是自然文本（可编辑）；开「显示字符编码」后变只读方格，按字节着色 + 标注 `U+XXXX · nB`。
- **当前 token 序列**：
  - **虚线灰块** = 半个字符的**裸字节**（如 `E4`），还没拼成完整字符；
  - **蓝色块** = 完整的单字节 ASCII 字符；
  - **彩色块** = 已拼成完整字符的 token，**同一个 token 永远同色**，一眼看出反复合并出的大单元；
  - **标点**统一弱化显示。
- **琥珀描边** = 下一步将被合并的相邻对（算法选频次最高的）。
- **霓虹青辉光环** = 刚刚这一步合并出的新 token。
- **当前词表**：只列**当前在用**（频次 > 0）的 token，按合并时间逆序，基础字节带「基础」标签。

自己试试输入 `你好你好你好你好`：24 个裸字节会一步步拼装，最终收成 2 个「你好你好」，
**压缩率 12×** —— 这就是 BPE 的核心魔法。

## 实现要点

- **字节 → 字符还原判定**：用 `new TextDecoder("utf-8", { fatal: true })`。完整 UTF-8 序列解码成功 →
  显示汉字；不完整 / 非法序列会抛异常 → 退回显示 hex。一个判定区分"完整字符"和"裸字节"。
- **token id 布局**：基础字节用字节值当 id（0–255），合并产物从 256 起递增 —— 与真实 GPT-2 词表布局一致。
- **超长文本不卡顿**：训练与统计跑全文，但 token 行 / 字符方格的渲染上限为 1500，超出折叠并提示
  「…还有 N 个 token（已折叠）」—— 所以贴一整篇 4 万字的论文也能流畅交互。
- 这是 **BPE 训练过程**（从语料学合并规则）的演示，不是用固定词表做编码的 Tiktokenizer。
